Post-Thumb Revisited crée automatiquement et affiche des miniatures pour les articles. De nombreuses fonctions et options permettant de modifier les thèmes sont disponibles.

== Description ==

Le plugin scan les articles à la recherche d'images. Il peut alors, effectuer les actions suivantes :

* Affichage dans la Boucle : affiche une miniature liée à un article.
* Affichage dans la sidebar : affiche des miniatures des derniers articles ou des miniatures d'articles choisis au hasard.

Toutes les miniatures sont créées dynamiquement, "à la volée", et sauvegardées. Il n'y a donc pas à s'embarasser à gérer les miniatures.
L'excellente bibliothèque javascript Highslide est incluse dans le plugin et permet d'ajouter des effets d'ouverture des plus esthétiques aux miniatures ou liens gérés par post-thumb revisited.


== Installation ==

1. Télécharger et dézipper le fichier.
2. Uploader le répertoire "post-thumb" dans le répertoire des plugins de votre blog WordPress.
3. Activer le plugin depuis le panneau d'administration.
4. Aller dans options->post thumb et configurer les options avant de l'utiliser.

Important : Vous devez configurer les options d'emplacement correctement sinon le plugin ne fonctionnera pas.


== Détails des options ==

Laissez les paramètres par défaut si vous hésitez. Lors de la sauvegarde des options, les principales options sont validées et un message d'erreur indique si un problème a été détecté.


= Options d'emplacement =

"Chemin du domaine sur le serveur", "Nom de domaine complet" and "Répertoire qui contiendra les thumbnails" doivent être configurés correctement ou le plugin ne fonctionnera pas.

* Chemin du domaine sur le serveur : Chemin absolu du site. Par exemple, /httpdocs ou /yourdomain.com. Utilisé pour retrouver l'emplacement des thumbnails sur le serveur. L'adresse http://yourdomain.com/images/pth/thumb_picture.jpg se traduirait en /httpdocs/images/pth/thumb_picture.jpg. Pas de '/' final.
* Nom de domaine complet : Nom de domaine complet, incluant le http://. Pas de '/' final.
* Répertoire qui contiendra les thumbnails : Définit le chemin relatif vers les thumbnails. Assurez-vous que le répertoire existe et peut être modifié (chmod 777). Pas de '/' final.
* Image par défaut : L'emplacement de l'image par défaut à utiliser si aucune référence d'image n'est trouvée. Entrez l'adresse relative depuis le site, par exemple, images/default.jpg
* Utiliser les meta-data ?
* Utiliser les noms de catégories : Si les noms de catégories sont utilisés, le nouveau nom constitué remplacera l'image par défaut des images et l'image par défaut des vidéos. Les images de catégorie doivent se trouver dans le même répertoire que l'image par défaut.
* Vérification de la configuration du système : le plugin va vérifier que la configuration du serveur est compatible avec post-thumb.
** Fichiers distants : vérifie que fopen est autorisé ou cURL disponible.
** PHP version : L'utilisation de fichiers distants requiert également php 4.3.0 ou supérieur.
** GD version : la bibliothèque GD 2 est requise.

== Mise en forme ==

Post-thumb renvoie du code html pur. Par conséquent, la mise en page peut être integralement gérée avec CSS.



== Fonctions de la Boucle ==

get_single_thumb () : Renvoie la miniature formatée de l'article en cours.
pt_the_excerpt() : Renvoie la miniature (+ Titre) + Excerpt.



== Fonctions de la Sidebar ==

get_single_thumb () : Renvoie la miniature formatée d'un article.
get_recent_thumbs () : Renvoie les miniatures des derniers articles.
get_random_thumb() : Renvoie la miniature d'un ou plusieurs articles au hasard.



== Constante supplémentaire pour les thèmes ==

POSTHUMB_INFRAME. Cette constante est définie chaque fois que vous appelez une page de votre blog. Elle est mise à 0 si l'appel est direct, et à 1 si l'appel est fait dans une fenêtre ouverte par highslide.

Cela vous permet de modifier l'apparence du thème dans les cadres popup.

Exemple :
		<?php if (POSTHUMB_INFRAME == 0) { ?>
			<?php get_sidebar(); ?>
		<?php } ?>

		Cela masquera la sidebar si la page est affichée dans un cadre popup.



== Paramètres ==

Chaque fonction peut être utilisée avec de nombreux paramètres qui permettent de contrôler l'affichage ou l'emplacement et le nom de sauvegarde des miniatures. Vous n'avez pas besoin de les utiliser à moins de vouloir utiliser des options différentes de celles par défaut définies dans l'écran d'administration.

Chaque paramètre doit être saisi avec sa valeur séparée par un "=" et séparé des autres paramètres par un '&'.

Exemple :
		<?php get_recent_thumb('WIDTH=100&HEIGHT=80&ROUNDED=1&LIMIT=12'); ?>



= Paramètres d'affichage =

* WIDTH: resize width. Overides default if greater than 0.
* HEIGHT: resize height. Overides default if greater than 0.
* HCROP: horizontal crop. Crops if greater than 0.
* VCROP: vertical crop. Crops if greater than 0.
* KEEPRATIO: if set to 1, image ratio is kept. Overides default if it exists.
* ROUNDED: creates thumbnails with rounded corners. Overides default if it exists.



= Paramètres supplémentaires d'affichage =

* TEXTBOX:  write a text in the bottom = 1. Default is 0.
* TEXT:  text to be written if TEXTBOX is set to 1.
* SHOWTITLE: display title, author or date below the thumbnail. Parameters: T, A, D.
* TITLE: choose wether content (C), excerpt (E) or title (T) is used in title tag of the thumbnail. Default is 'T'.
* MYCLASSHREF: output class name in html `<a class="myclasshref" href=...>`
* MYCLASSIMG: output class name in html `<img class="myclassimg" href=...>`
* LB_EFFECT: use highslide to display image or link.
* CAPTION: display or not the caption for pictures.



= Paramètres de sauvegarde =

* ALTAPPEND: text to append to create thumbnail name. Overide default if exists.
* BASENAME: force generation of thumbnail and use generic name. Default is 0. Used in the random function.
* SUBFOLDER: name of the subfolder to save thumbnails in. Only one level, no wrapping '/'.



= Paramètres de thème =

* USECATNAME: choose if category default image should used or not. Override default if exists.
* SHOWPOST: force Highslide to expand on post if set to 1. Default is 0 (expand on image or post)



= Paramètres de get_recent_thumb =

* LIMIT: number of posts to display
* OFFSET: skip posts by the given number. Default is 0.
* CATEGORY: categories to get posts from, or categories to exclude from search.


= Paramètres de get_random_thumb =

* LIMIT: number of posts to check to return a random image. Default is 5.
* RLIMIT: number of posts to display.
* CATEGORY: category to search random from.



== Addtionnal functions ==

In addition to the basic function, Post-Thumb comes with two plugins which can be activated separately and adds new functions.
= Post-thumb-post-library =
* wordTube media tags: will wrap each video or mp3 with a thumbnail.
* youtube specific tag: will wrap youtube player with a thumbnail.
* automatic thumbnailing in posts: the plugin will scan for the entire post and will thumbnail all pictures which has the attribute rel="thumb" on it.
* automatic expand effect: the plugin will scan for the entire post and will add the Highslide expansion effect to each thumbnail link to a url.

== Formattage des articles ==

= Images et miniatures =

Si vous insérer une image avec une miniature dans un article, par exemple, avec Image Manager, vous pouvez choisir d'afficher l'image en utilisant Highslide.
Cela peut être activé/désactivé dans Options->Options Highslide->Utiliser Highslide dans les articles.

Si vous ajoutez rel="thumb" au lien d'une image dans un article, le plugin créera automatiquement la miniature et la placera dans l'article.
Cela peut être activé/désactivé dans les options et l'aspect des miniatures peut être défini dans les options. Tout cela se trouve dans les Options supplémentaires.

= Media wordTube =

Les media WordTube peuvent être détectés et affiché avec des miniatures. Le media sera joué dans un cadre popup en cliquant sur la miniature.
Voir les Options pour ajuster les paramètres.



= Playlist wordTube =

Post-thumb revisited fournit son propre système de playlist pour wordTube.

* [PTPLAYLIST=(n,n,n)|WIDTH=xxx|HEIGHT=xxx|MP3/FLV]
MP3/FLV will select either one of the two. Default is all (none selected). Playlist with 0 will select all media.



== Insertion de vidéos Youtube ==

Les vidéos Youtube peuvent être insérées dans les articles en utilisant une balise simple. The tag will display a thumbnail and the video will be viewed in an expanded frame when clicking on it.

* [YOUTUBE=(video ID) title=(video title)]




== Widgets ==

Widgets are gathered in the Post-thumb widgets plugin. It needs to be activated. If your theme is widget ready, then the followings wil be available:

Fonctions supportée :

* pt-random : returns thumbnails from random posts
* pt-recent : returns thumbnails from most recent posts
* pt-categories : list categories. Same as wp_list_categories but adds a popup link.
* pt-bookmarks : list bookmarks. Same as wp_list_bookmarks but opens link in a popup frame.
* pt-wordTube : to display a thumbnail linked to a wordTube media in the sidebar

Fonctions non-supportées :

* pt-slideshow
* pt-news
